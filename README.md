# Физика: Задачи и учебные материалы — приложение для изучения физики

Учебный проект по веб-разработке

## Описание проекта

Веб-приложение на Django, позволяющее изучать физику с помощью учебных статей, решения задач с автопроверкой, прохождения контрольных работ и режима «Мозговой штурм» (случайный набор задач по выбранным темам).

Проект выполнен в рамках учебного задания.

**Основной функционал:**

- Регистрация и аутентификация пользователей.
- Учебник: статьи по темам (создаются администратором). В конце статьи — список задач по теме и ссылка на контрольную.
- Задачи: список задач с фильтром по теме. Любой зарегистрированный пользователь может создать свою задачу. При решении показываются похожие задачи из той же темы.
- Контрольная работа: набор из 2+ задач по одной теме (собирается администратором). После прохождения отображается результат с подсветкой правильных/неправильных ответов.
- Мозговой штурм: пользователь выбирает количество задач и темы. Система генерирует случайный набор так, чтобы из каждой выбранной темы была хотя бы одна задача.
- Все данные хранятся в SQLite.
- Для отображения формул используется MathJax (LaTeX).

## Быстрый запуск (Windows)


1. Скачайте или клонируйте репозиторий.
2. В корне проекта (там, где лежит `manage.py`) дважды кликните по файлу **`RUN.bat`**.
3. Следуйте инструкциям в консоли:
   - Если суперпользователь отсутствует, скрипт предложит его создать.
   - Если суперпользователь уже есть, можно пропустить создание нового.
4. После запуска сервера откройте браузер по адресу `http://127.0.0.1:8000`.

> **Примечание:** скрипт автоматически создаст виртуальное окружение, установит зависимости из `requirements.txt`, применит миграции, загрузит тестовые данные (статьи, задачи, контрольные) и запустит сервер. Для остановки сервера нажмите `Ctrl+C`.

## Подробная ручная установка

Для ручного запуска выполните следующие шаги:

1. Клонировать репозиторий:

   git clone https://github.com/Carcosa-civ/physics_tasks.git
   cd physics_tasks
Создать виртуальное окружение и активировать его:

python -m venv venv
source venv/bin/activate   # для Linux/Mac
venv\Scripts\activate      # для Windows
Установить зависимости:

pip install -r requirements.txt
Создать файл .env (см. раздел «Корректное хранение секретов») и поместить в него SECRET_KEY и DEBUG.

Применить миграции:

python manage.py makemigrations
python manage.py migrate
(Опционально) Загрузить тестовые данные (статьи, задачи, контрольные работы, суперпользователя admin/admin):

python manage.py load_test_data
Запустить сервер разработки:

python manage.py runserver
Открыть в браузере: http://127.0.0.1:8000

Корректное хранение секретов и токенов
Для загрузки секретов как переменных окружения используется библиотека python-dotenv.

Все секретные токены (например, SECRET_KEY) должны быть спрятаны в коде за импортом. Для этого создайте в корне репозитория файл .env и добавьте в него пары «переменная-значение».

Файл .env нужно обязательно поместить в .gitignore!

Пример настройки для данного проекта
Установите python-dotenv (обычно уже есть в requirements.txt):

bash
pip install python-dotenv
Создайте в корне проекта (рядом с manage.py) файл .env со следующим содержимым:

text
SECRET_KEY=django-insecure-ваш-уникальный-ключ
DEBUG=True
В physics_tasks/settings.py уже настроена загрузка переменных:

python
from dotenv import load_dotenv
import os
from pathlib import Path

BASE_DIR = Path(__file__).resolve().parent.parent
dotenv_path = os.path.join(BASE_DIR, '.env')
load_dotenv(dotenv_path)

SECRET_KEY = os.getenv('SECRET_KEY')
if not SECRET_KEY:
    raise ValueError("SECRET_KEY не задан в файле .env")

DEBUG = os.getenv('DEBUG') == 'True'
Теперь секретный ключ хранится вне кода, что повышает безопасность проекта.

Проверка качества кода
Для проверки кода используется pylint с плагином pylint-django. Запустите скрипт pylint.bat (Windows) или выполните вручную:

bash
pylint --rcfile=.pylintrc core
Ожидаемая оценка: не ниже 9.0/10.
